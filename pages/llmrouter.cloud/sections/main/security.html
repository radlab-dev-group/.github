Security-First Architecture

LLM Router is built with data protection at its core, suitable for organizations of any size where data security matters. Every request is automatically protected through a two-layer security system: guardrails that validate content safety before processing, and masking pipelines that anonymize sensitive data before transmissionâ€”whether to your local models (vLLM, Ollama, LM Studio) or cloud providers. Keep sensitive data secure within your infrastructure while ensuring compliance with GDPR, HIPAA, SOC2, and other regulatory frameworks.

Guardrails - Content Safety Layer

  • Multi-stage guardrail pipelines for content validation and safety checks
  • NASK Guard integration for Polish language content safety (PL-Guard model)
  • Extensible plugin architecture for custom guardrail implementations
  • Request blocking for unsafe or prohibited content before LLM processing
  • Configurable guardrail chains with sequential validation stages

Data Masking - PII Protection Layer

  • Automatic PII detection and masking (GDPR/CCPA/HIPAA compliant)
  • Support for international identifiers: SSN, tax IDs, national IDs, business registration numbers
  • Checksum validation for detected identifiers ensuring accuracy
  • Extensible masking engine with custom rule support for industry-specific requirements
  • Multi-plugin masking pipelines with sequential data transformation
  • Enforced anonymization with configurable strategies across entire payload
