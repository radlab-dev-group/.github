Monitorowanie i Health

Wbudowany system monitorowania zapewniający niezawodność, obserwowalność i proaktywne zarządzanie infrastrukturą.

🫀

KeepAliveMonitor

Periodically pings model endpoints to keep them warm. Eliminates cold-start latency for large models.

  • Configurable ping interval
  • Per-model endpoint tracking
  • Automatic host registration in Redis
  • Reduces first-request latency by ~10-30s
LLM_ROUTER_KEEPALIVE_MODEL_MONITOR_INTERVAL_SECONDS=10
🏥

ProviderMonitor

Continuously probes model provider endpoints to track availability. Integrates with the first_available load balancing strategy.

  • Automatic health checking
  • Redis-backed availability cache
  • Fast failover to healthy providers
  • Circuit breaker pattern
LLM_ROUTER_PROVIDER_MONITOR_INTERVAL_SECONDS=5
📡

ServicesMonitor

Monitors external guardrail and security services (e.g. Sojka-Guard, NASK Guard). Ensures the security pipeline is fully operational.

  • External service health checks
  • Guardrail pipeline readiness
  • Redis-synced availability
  • Automatic pipeline switching
LLM_ROUTER_SERVICES_MONITOR_INTERVAL_SECONDS=5
📊

Prometheus Integration

Włącz LLM_ROUTER_USE_PROMETHEUS=1, aby uzyskać metryki w czasie rzeczywistym:

● request_rate
● request_latency
● error_rate
● provider_status
● keepalive_status
● model_latency
