Usage Workflows

Common tasks and operations

Config Manager: First-Time Setup

  • Access http://localhost:8081/ – redirected to /setup (no users exist)
  • Create initial admin user with username and password
  • Login with admin credentials
  • Default project "default_project" automatically created
  • Create first configuration via "New Config" button
  • Add models and providers via visual editor
  • Mark models as active, export to JSON, activate configuration

Config Manager: Adding Users (Admin Only)

  • Navigate to Admin → Users
  • Click "Add User", enter username/password/role
  • New user gets automatic default project
  • Edit user role or reset password via edit form
  • Block/unblock users to disable/enable access

Config Manager: Version Control

  • Every config change (model add/delete, provider update, active model change) creates automatic snapshot
  • View version history on config detail page
  • Restore to previous version via "Restore" button
  • Add optional note when saving changes
  • Version number auto-increments

Anonymizer: Text Anonymization

  • Access http://localhost:8082/anonymize/
  • Paste text into textarea
  • Select algorithm: fast (default), genai (requires model), priv (not available)
  • Click "Anonymize" – result displayed inline
  • FastMasker: regex-based, instant, no external calls
  • GenAI: forwards to LLM Router /api/anonymize_text_genai endpoint

Anonymizer: Interactive Chat

  • Access http://localhost:8082/anonymize/chat
  • Select model from dropdown (populated via /anonymize/models proxy)
  • Choose anonymization: no_anno, fast, genai
  • Type message, click "Send"
  • Request forwarded to LLM Router /v1/chat/completions with anonymize flag
  • Response rendered inline below message box
