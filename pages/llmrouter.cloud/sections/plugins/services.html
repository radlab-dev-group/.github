Services & Integration

Standalone Flask services expose ML models as HTTP endpoints, wired dynamically via environment variables.

🛡️

Guardrails API

llm-router-services

Unified Flask app exposing guardrail endpoints. Routes are dynamically registered based on *_ENABLED environment variables.

Endpoints

POST /api/guardrails/nask_guard NASK-PIB safety check
POST /api/guardrails/sojka_guard Sojka safety check
# Enable NASK Guard
export LLM_ROUTER_NASK_PIB_GUARD_ENABLED=1
./run_services.sh
🔒

Maskers API

llm-router-services

Token-classification based PII anonymizer with in-memory caching. Loads models once at startup, serves over HTTP.

Endpoints

POST /api/maskers/pii_masker PII detection & redaction
# Payload (array of texts supported)
{"payload": "NIP: 1234567890"}
# → anonymized + mappings

Plugin Registry

llm-router-plugins

Central registry pattern for lazy plugin discovery and pipeline orchestration. Three independent registries: Masker, Guardrail, and Utils.

Features

✓ Automatic discovery via register()
✓ Pipeline instantiation by identifier
✓ Graceful error handling per plugin
✓ Environment-driven activation
class MyGuard(HttpPluginInterface): name = "my_guard"
def apply(self, payload): return True, {}
