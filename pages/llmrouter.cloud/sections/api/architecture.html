System Architecture

Modular layered design with clear separation of concerns

01. Execution Stack

REST Interface core/server.py
Flask/Gunicorn entry point. Handles raw HTTP ingress and automatic route registration.
Engine Core core/engine.py
Orchestrator that initializes components, discovers endpoints, and binds routes.
Plugin Pipeline plugins/*
Three extensible pipelines: MaskerPipeline, GuardrailPipeline, UtilsPipeline. Each plugin implements HttpPluginInterface.
Endpoint Logic endpoints/*
Proxy layer (EndpointWithHttpRequestI) connecting specific API logic with core services.
Load Balancing base/lb/*
Strategy pattern implementation (Weighted, Round-Robin) via ProviderChooser. Balanced, weighted, first_available, first_available_optim strategies.
Provider Layer model_handler.py
Resolves abstract model names to concrete API providers and configurations. Supports OpenAI, Ollama, vLLM, LM Studio, llama.cpp, Anthropic, Google.
HTTP I/O httprequest.py
Low-level execution handling, retries with exponential backoff, streaming response management with SSE.
Monitoring monitoring/*
KeepAliveMonitor, ProviderMonitor, ServicesMonitor — Redis-backed health tracking and Prometheus metrics.

02. Core Components

⚡ FlaskEngine
The heart of the application. Bootstraps the environment and loads plugins.
🏗️ EndpointI
Base abstract class enforcing validation, logging, and error handling standards.
⚖️ ProviderChooser
Facade for LB strategies. Delivers get_provider() unified interface.
🗺️ ModelHandler
Manages lifecycle of providers and mapping of virtual models to endpoints.
📡 HttpRequestEx
Robust HTTP client wrapper with automatic backoff and stream processing.
🔌 ApiDispatcher
Routes requests based on API type (OpenAI vs Ollama vs vLLM formats).
🔒 MaskerPipeline
Detects and masks PII data (PESEL, NIP, emails, etc.) with 25+ rules and GPG auditing.
🛡️ GuardrailPipeline
Content safety validation pipeline with Bielik-Guard (Sojka) and NASK Guard integration.
🔧 UtilsPipeline
Utility plugins: semantic_routing, langchain_rag, translate, generate_questions.
