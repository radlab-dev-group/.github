Monitoring & Metrics

Built-in monitoring system and Prometheus integration

Monitoring Components

🫀

KeepAliveMonitor

Periodically pings model endpoints to keep them warm. Eliminates cold-start latency.

  • Configurable ping interval
  • Per-model endpoint tracking
  • Redis-backed host registration
  • Reduces first-request latency by ~10-30s
🏥

ProviderMonitor

Continuously probes model provider endpoints to track availability.

  • Automatic health checking
  • Redis-backed availability cache
  • Fast failover to healthy providers
  • Circuit breaker pattern
📡

ServicesMonitor

Monitors external guardrail and security services (Sojka-Guard, NASK Guard).

  • External service health checks
  • Guardrail pipeline readiness
  • Redis-synced availability
  • Automatic pipeline switching

Prometheus Integration

When LLM_ROUTER_USE_PROMETHEUS=1

  • /metrics endpoint exposed in Prometheus format
  • Request counts tracked per endpoint and model
  • Latency histograms for response time distribution
  • In-progress gauges for concurrent requests
  • Error counters for failure tracking
  • Provider status gauge for health tracking
  • KeepAlive status for model readiness
# Example: Prometheus scrape config
scrape_configs:
  - job_name: 'llm-router'
    static_configs:
      - targets: ['localhost:8080']
    metrics_path: '/metrics'
    scrape_interval: 15s

Auditor

AnyRequestAuditor

  • Records audit logs for masking and guardrail operations.
  • Logs contain endpoint name, audit type, timestamps, and payload snapshots.
  • Configured via environment variables MASKING_WITH_AUDIT and GUARDRAIL_WITH_AUDIT_REQUEST/RESPONSE.
