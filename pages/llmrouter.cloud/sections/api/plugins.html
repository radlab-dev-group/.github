Extendable pipeline architecture for safety, privacy, and utility

Security Layer

Guardrail Pipelines

First line of defense. Validates content safety before processing using sequential plugins to block unsafe requests.

  • Pipeline Architecture: Sequential validation plugins
  • Bielik-Guard (Sojka): Advanced safety classifier for Polish (Apache-2.0)
  • NASK Guard: PL-Guard model integration (CC BY-NC-SA 4.0)
  • Custom Plugins: Extend via HttpPluginInterface
# Configure guardrail pipeline
export LLM_ROUTER_FORCE_GUARDRAIL_REQUEST=1
export GUARDRAIL_STRATEGY_PIPELINE_REQUEST="sojka_guard,nask_guard"

# 1. Sojka Guard validates content (HATE, CRIME etc.)
# 2. NASK Guard applies safety rules
# Unsafe content → Request Blocked 🚫
🔒
Privacy Layer

Masking Pipelines

Automatically detects and anonymizes sensitive PII data before transmission to LLM providers.

  • FastMasker: High-performance PII detection (25+ rules)
  • Rule Types: PESEL, NIP, KRS, REGON, Emails, NRB, Credit Cards, VIN
  • Validation: Checksum validation & GPG encrypted auditing
# Enable masking with audit
export LLM_ROUTER_FORCE_MASKING=1
export LLM_ROUTER_MASKING_WITH_AUDIT=1

# Input: "NIP: 1234567890"
# Output: "NIP: {{NIP}}"
🔧
Utility Plugins

Utils Pipeline

Ready-to-use utility plugins for common AI workflows: semantic routing, RAG, translation, and question generation.

  • semantic_routing: Context-aware routing to optimal providers
  • langchain_rag: RAG with LangChain and FAISS integration
  • translate: Batch translation with configurable target languages
  • generate_questions: Auto-generate questions from input texts
# Enable utility plugins
export UTIL_PLUGINS="semantic_routing,langchain_rag,translate"

# Semantic routing auto-selects
# the best provider based on context

Framework

Plugin Architecture

Unified interface for Guardrails, Maskers, and Utils. Easily extend the system with your own logic.

  • PluginInterface: Base class with apply(payload)
  • Registry Pattern: Automatic discovery & registration
  • Graceful Degradation: Safe error handling for plugins
  • Three Pipelines: Masker → Guardrail → Utils (sequential)
class MyGuard(HttpPluginInterface):
name = "my_guard"

def apply(self, payload):
is_safe = check(payload)
